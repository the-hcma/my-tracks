# Copilot Agent Directives

**Important**: These directives must be followed for all work on this project.

## Server Management

### Always Use my-tracks-server Script

**CRITICAL**: When starting the development server, ALWAYS use the `my-tracks-server` script:

```bash
./scripts/my-tracks-server
```

**Never use** `python manage.py runserver` directly. The `my-tracks-server` script ensures:
- Virtual environment is activated
- Port 8080 is used consistently
- `uv` is in PATH
- Proper error handling
- We continuously test the script itself
- Dual logging mode with `--console` flag (outputs to both console AND file)

**Rationale**: By always using the `my-tracks-server` script, we ensure it stays working and catches any issues with the setup process.

## Package Management

- **ONLY use `uv`** for Python package management
- Install dependencies: `uv sync` (or `uv sync --all-extras` for dev)
- Never use `pip` directly

## Script Convention

### Shell Scripts
- All shell scripts MUST be created **without the `.sh` extension**
- Examples: `setup`, `my-tracks-server`, `deploy` (not `setup.sh`, `my-tracks-server.sh`, etc.)
- Make executable: `chmod +x scriptname`
- Use shebang: `#!/usr/bin/env bash`

### Python Scripts
- All standalone Python scripts MUST be created **without the `.py` extension**
- Examples: `generate-tail`, `install`, `verify-setup` (not `generate_tail.py`, `install.py`, etc.)
- Make executable: `chmod +x scriptname`
- Use shebang: `#!/usr/bin/env python3`
- **MUST include auto-venv activation** (see AGENTS.md for pattern)
- **MUST use Typer** for CLI argument parsing
- Rationale: Cleaner command-line interface, scripts work without manual venv activation

## Git and Branch Management

> Stacking SSOT: [`.cursor/rules/stacking-tool.mdc`](./.cursor/rules/stacking-tool.mdc)
> (canonical skill in repository-helpers). Cheat sheet: [docs/GH-STACK.md](./docs/GH-STACK.md).

### Session Start — Initialize via start-development

**CRITICAL**: At the start of every session, before any other work, run both commands in order:

```bash
~/work/ai/repository-helpers/scripts/dev/start-development --refresh
~/work/ai/repository-helpers/scripts/dev/start-development
```

- **`--refresh`** (first): marker-aware sync (`.github/stacking-tool` → `gh-stack`), prunes merged worktrees and branches, pulls latest main, and ensures the background service is running. Exits immediately — it does **not** create a worktree.
- **Second invocation** (required): creates the stack worktree — plain `start-development` (prompts for a name) or `--worktree <stack-name> --no-interactive`.
- Both commands are required: `--refresh` checks/starts the service; the second invocation creates the worktree.

### MANDATORY: Use `gh stack` for Version Control

**CRITICAL**: Use `gh stack` (not Graphite `gt`) for stacked PR workflow. Marker: `.github/stacking-tool` = `gh-stack`.

**Common commands**:
- ✅ First layer: `gh stack init <stack>/<topic>` then commit
- ✅ Next layer: `gh stack add <stack>/<next>`
- ✅ Submit: `~/work/ai/repository-helpers/scripts/dev/submit-stack` or `gh stack submit --auto --open --remote origin`
- ✅ View: `gh stack view --json`
- ✅ Sync: `gh stack sync --remote origin`
- ✅ Navigate: `gh stack up` / `gh stack down`

**Merge**: `gh pr merge <pr> --auto --squash` — never use `merge-it` / `merge-mq` labels.

**Rationale**: GitHub Stacked PRs + merge queue keep review diffs focused and align with repository-helpers.
## Testing

- Always run tests before committing: `uv run pytest`
- Achieve 90%+ code coverage
- Include randomized tests for all implementations
- **All test assertions MUST use PyHamcrest** — never naked `assert` statements:
  - ✅ `assert_that(result, equal_to(42))`
  - ✅ `assert_that(value, is_(not_none()))`
  - ❌ `assert result == 42`
  - ❌ `assert value is not None`
- **All imports MUST be at module level** — no local/lazy imports inside test functions or methods:
  - ✅ `from app.models import Device` at the top of the file
  - ❌ `from app.models import Device` inside a test function body
  - Rationale: Local imports hide dependencies and complicate patching
